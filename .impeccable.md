## Design Context

### Users
Professional traders, portfolio managers, and fintech operations teams. They use this dashboard during active trading sessions — often for hours at a time. The job to be done: monitor positions, evaluate signals, execute trades, and manage risk with zero ambiguity. Every second of confusion costs money.

### Brand Personality
**Precise. Trustworthy. Calm.**

The interface communicates quiet authority. It doesn't shout, decorate, or gamify. It presents data with surgical clarity and gets out of the way. Think Bloomberg Terminal restraint meets Linear's stripped-back elegance.

### Aesthetic Direction
- **Visual tone**: Minimal, precise, data-first. Every element earns its place or gets removed.
- **References**: Linear, Stripe Dashboard, Bloomberg Terminal (restraint, not complexity)
- **Anti-references**: Purple gradients, rounded bubbly cards, decorative borders, box shadows for flair, generic dashboard templates, gamification aesthetics, ornamental icons, excessive colour variety
- **Theme**: Dark mode primary. Light mode support secondary. Dark interfaces reduce eye strain during long trading sessions.
- **Color**: Maximum 3 hues. One neutral base, one accent for primary actions, profit/loss pair (green/red). Colour means data — never decoration.
- **Typography**: Clean sans-serif for UI, monospace for all numerical values (prices, quantities, percentages). Tabular numbers for column alignment.

### Design Principles

1. **Subtract, don't add.** If an element doesn't help the trader make a decision or execute a trade, remove it. White space and alignment do the work, not chrome.
2. **Data is the interface.** Numbers, charts, and status indicators are the UI. Everything else is support structure that should be invisible.
3. **Colour communicates value.** Green = profit/up, red = loss/down. These are the only saturated colours. Everything else is neutral. If something is coloured, it's communicating state.
4. **Mobile is not an afterthought.** A trader checks positions from their phone. Touch targets minimum 44px. Layouts collapse gracefully. No horizontal scroll on any breakpoint.
5. **Consistency over cleverness.** One spacing scale, one type scale, one component style. No per-component exceptions. The entire application is one cohesive surface.

### Technical Constraints
- React (Next.js 16, App Router)
- Tailwind CSS 4 with CSS variables
- Shadcn UI component library (customizable)
- Recharts for data visualization
- Responsive: 320px through 2560px
- WCAG AA minimum contrast ratios
- `prefers-reduced-motion` respected — disable all non-essential animation
- No Inter font — use a clean sans-serif + monospace pairing appropriate for financial data
- No box shadows for visual depth — use borders (1px, subtle) or whitespace
- No card-in-card patterns — flat sections with subtle dividers or whitespace only
